# devHub

A single-page **Angular 21** hub that bundles several self-contained feature pages — auth, an admin/shop back office, real-time chat, a 3D Rubik's cube with an autosolver, a "Now Playing" cinema board, and more — behind a shared navbar and a token-based login.

State is built on **Angular signals**, the UI uses **Angular Material**, and the app is **SSR-ready** (Angular SSR + Express).

## Features

| Page              | Route                      | What it does                                                                                         |
|-------------------|----------------------------|------------------------------------------------------------------------------------------------------|
| **Login**         | `/login`                   | Token-based authentication; auth state is exposed as a signal from `AuthService`.                    |
| **Dashboard**     | `/dashboard`               | Landing overview with charts (ApexCharts).                                                           |
| **Admin**         | `/admin`                   | Manage users, products and orders.                                                                   |
| **Shop**          | `/shop`                    | Product listing / ordering.                                                                          |
| **Order History** | `/order-history`           | Past orders for the signed-in user.                                                                  |
| **Entity Form**   | `/entity-form/:type[/:id]` | Generic create/edit form reused across admin entities.                                               |
| **Chat**          | `/chat`                    | Real-time group chat over **SignalR**.                                                               |
| **Rubik's Cube**  | `/rubiks-cube`             | Interactive 3D cube (**Three.js**) with a **Kociemba two-phase autosolver** running in a Web Worker. |
| **Cinema**        | `/cinema`                  | "Now Playing" film board generated by a scraper (see [Cinema data](#cinema-data)).                   |
| **Spotify**       | `/spotify`                 | Spotify-themed feature page.                                                                         |
| **Feedback**      | `/feedback`                | Feedback submission form.                                                                            |

All feature routes are protected by `AuthGuard`; unauthenticated users are redirected to `/login`.

## Tech stack

- **Angular 21** (signals, standalone-free `NgModule` setup) + **Angular CLI**
- **Angular Material** + CDK
- **Three.js** — 3D Rubik's cube rendering
- **@microsoft/signalr** — real-time chat
- **ApexCharts** (`ng-apexcharts`) — dashboard charts
- **Angular SSR + Express** — server-side rendering
- **node-html-parser** — cinema scraper (dev tooling)

## Prerequisites

- **Node.js** (LTS) and npm
- A running **backend API** for auth and data (the app expects it at `https://localhost:5000/api` — see `src/services/auth.service.ts`)

## Getting started

```bash
npm install
npm start          # ng serve → http://localhost:4200/
```

The dev server reloads automatically on source changes.

## Available scripts

```bash
npm start                    # ng serve (dev server)
npm run build                # production build → dist/
npm run watch                # rebuild on change (development config)
npm test                     # unit tests via Karma
npm run scrape:cinema        # refresh the Cinema page's film data
npm run serve:ssr:devHub-app # serve the SSR build (after npm run build)
```

## Cinema data

The Cinema page reads from `src/app/components/cinema/films.data.ts`, which is **generated** by a scraper rather than hand-edited:

```bash
npm run scrape:cinema
```

This runs `tools/scrape-cinema.mjs`, which fetches current listings, parses them with `node-html-parser`, and overwrites `films.data.ts` (it stamps a `SCRAPED_AT` timestamp). Re-run it whenever you want to refresh "Now Playing".

## Project structure

```
src/
  app/
    components/      # one folder per feature page (see table above)
    app-routing.module.ts
    app.module.ts
  services/          # auth, session, notifications, http interceptor, guards, …

  scrape-cinema.mjs  # generates cinema/films.data.ts
```
